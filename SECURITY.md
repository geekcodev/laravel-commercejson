## Сообщение об уязвимости

Если вы обнаружили уязвимость безопасности, пожалуйста, отправьте письмо на адрес geekco@yandex.ru.

Все уязвимости безопасности будут оперативно рассмотрены и исправлены.

Пожалуйста, укажите:

- Описание уязвимости
- Шаги для воспроизведения
- Потенциальные последствия
- Предлагаемое исправление (если есть)

## Лучшие практики безопасности

### Для пользователей

1. **Храните API-токен в безопасности**
    - Никогда не коммитьте токены в систему контроля версий
    - Используйте переменные окружения
    - Периодически обновляйте токены

2. **Используйте HTTPS**
    - Всегда используйте HTTPS для подключения к API
    - Проверяйте SSL-сертификаты

3. **Ограничивайте права доступа**
    - Используйте минимально необходимые права
    - Регулярно обновляйте учётные данные

4. **Мониторьте логи**
    - Следите за неудачными попытками аутентификации
    - Отслеживайте необычную активность API

### Для контрибьюторов

1. **Никогда не коммитьте чувствительные данные**
    - API-токены
    - Пароли
    - Закрытые ключи

2. **Используйте .gitignore**
    - Файлы конфигурации с секретами
    - Файлы .env
    - Файлы баз данных

3. **Проверяйте код на уязвимости**
    - SQL-инъекции
    - XSS-уязвимости
    - CSRF-защита

## Процесс обработки уязвимостей

1. **Получение отчёта**
    - Отправка на email: geekco@yandex.ru
    - Ответ в течение 48 часов

2. **Анализ уязвимости**
    - Проверка воспроизводимости
    - Оценка серьёзности
    - Определение затронутых версий

3. **Исправление**
    - Разработка патча
    - Тестирование исправления
    - Выпуск обновления

4. **Публикация**
    - Уведомление в CHANGELOG
    - Обновление SECURITY.md
    - Благодарность автору отчёта (по желанию)

## Типы уязвимостей

### Критические

- Удалённое выполнение кода
- SQL-инъекции
- Аутентификация без прав доступа

### Высокие

- XSS (Cross-Site Scripting)
- CSRF (Cross-Site Request Forgery)
- Раскрытие чувствительных данных

### Средние

- Недостаточная валидация входных данных
- Недостатки авторизации

### Низкие

- Информационные утечки
- Недостатки логирования
